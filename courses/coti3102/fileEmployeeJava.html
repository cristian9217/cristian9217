Final Homework: File of Employees

COTI 3102 - Algorithms and Program Development II

IDE

Requirments

  • Eclipse IDE for Java Developers

Files

For this project:

  • employee.txt

Problem

1. In package edu.uprb.company place an enumerated type named Department with the following values and corresponding toString method:

  • SALE – for Budget
  • MKTG – for Marketing
  • HMRS – for Human Resources
  • FINA – for Finance
  • INTE – for Information Technology

2. In package edu.uprb.company place an abstract class named Employee that holds the following data for a company employee:

  • id – an integer between 1000 and 9999. This should be a final field.
  • name – a string with one uppercase letter and, at most, nine more lowercase letters; default value = "John".
  • lastName – a string with one uppercase letter and, at most, nine more lowercase letters; default value = "Doe".
  • department – a Department; default value = FINA.
  • hours – hours worked during the week an integer between 0 and 60; default value = 40.
  • payRate – a floating-point number between 7.00 and 50.00; default value = 8.25.

3. Since a class diagram for the classes in package edu.uprb.company was not provided, you must generate one using the ObjectAid UML Explorer for Eclipse. Save this diagram as file EmployeeClassDiagram.jpg in your project.

4. In package edu.uprb.company place an application EmployeePayroll that that reads a file of employees (employee.txt) and create another file that writes their data with the appropriate format, including their weekly salary.

Solution

Conclusion

Finally, Java's support for abstract classes, enumerations, and I/O files flexibility the language's versatility and popularity in software development. Abstract classes allow the development of common functionality that may be utilized by several classes, allowing each classes to define unique activities while maintaining code consistency. Enumerations allow you to declare a specified set of values, making it easier to manage and validate input while also reducing code errors and problems. I/O files are a vital part of software development, and Java's extensive set of classes and methods provides developers with the tools they need to manage complex data storage and retrieval activities.
